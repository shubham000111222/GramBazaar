# Non-Functional Requirements

## Performance
- System should handle multiple users simultaneously
- Fast response time for product search and checkout

## Security
- Secure authentication and authorization
- Encrypted payment transactions

## Usability
- Simple and user-friendly interface
- Support for users with low digital literacy

## Reliability
- System should be available 24/7
- Minimal downtime

## Scalability
- System should support future growth in users and products
